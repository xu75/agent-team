# Thread UI 视觉回归（快照验收）

目的：在 `project-*` → `thread-*` 选择器重命名阶段，确保样式绑定未回归。

## 固定步骤

1. 启动 UI：

```bash
npm run ui
```

2. 打开 `http://127.0.0.1:4173`，固定到以下状态后截图：
- 左侧 Thread 下拉已展开
- 至少 1 个已归档 Thread（显示“已归档”标签）
- 搜索框为空，任务分组可见

3. 生成两张快照：
- `before-thread-css.png`（改动前）
- `after-thread-css.png`（改动后）

4. 对比规则：
- Thread 下拉容器位置与宽度一致
- 条目 hover/active 背景色一致
- “已归档”标签字号、圆角、底色一致
- 左侧分组标题（Thread 名/计数）对齐一致

## 通过标准

- 上述 4 项均无可见差异，或差异仅为预期文案变化。
- 若出现布局错位/颜色丢失/标签消失，视为不通过。
